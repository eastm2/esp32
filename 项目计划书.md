## 一、项目背景与目标
### 1.1 项目背景
人体活动识别（HAR）是智能穿戴、健康监护、跌倒检测等领域的核心技术。本实训依托前置《物联网技术》课程掌握的MPU6050/HMC5883L传感器知识，结合《模式识别与统计学习》的统计决策、特征工程、分类器设计等理论，完成从传感器原始数据采集到模式识别算法落地的全链路工程实践，最终实现一个可区分6-8类日常活动、支持ESP32边缘推理的可穿戴识别系统。

### 1.2 核心目标
1. **基础目标**：完成≥6类人体活动（静坐、站立、步行、跑步、上楼、下楼）的自采数据集构建，实现标定→预处理→特征工程→分类→评估的完整模式识别链路，被试独立（LOSO）评估下整体准确率≥80%，达到实训合格标准。
2. **进阶目标（冲80+）**：完成多传感器PCB扩展板设计、打样与焊接验证（I2C可读取传感器），满足总评80分以上的硬性门槛；实现最小风险贝叶斯决策（跌倒检测场景）、模型轻量化与ESP32实时推理Demo。
3. **创新目标**：探索特征融合与时序建模（HMM）提升易混淆类（如上楼/下楼）识别率，形成可复现、可归档的课程级技术报告。

## 二、项目技术路线
本项目采用“硬件-数据-算法-评估”并行推进的全链路工程架构，核心流程如下：
```
[硬件层] → [数据层] → [特征层] → [模型层] → [评估与落地]
PCB扩展板设计  → 标定后数据采集  → 时域+频域+时频特征  → 贝叶斯/LDA/集成学习  → LOSO评估/显著性检验
  ↓               ↓                ↓                   ↓                  ↓
KiCad绘图      六位置/椭球拟合    Filter/Wrapper选择  超参网格搜索         ESP32边缘部署
嘉立创打样     防泄漏数据划分     PCA/LDA降维         代价敏感决策         实时Demo验证
```

## 四、关键技术难点与解决方案
| 难点 | 风险等级 | 解决方案 |
|------|----------|----------|
| PCB打样延迟错过D6-D7焊接窗口 | ⭐⭐⭐⭐⭐ | 严格执行D3中午前下单红线；提前联系嘉立创客服加急；若延迟，先用杜邦线方案推进算法，板到后无缝切换 |
| 磁力计受ESP32 WiFi干扰导致标定失效 | ⭐⭐⭐⭐ | 布局时HMC5883L远离天线≥15mm；标定与采集时关闭WiFi；若干扰仍存在，暂时屏蔽磁力计特征，优先保障加速度/陀螺仪数据质量 |
| 上楼/下楼易混淆导致整体精度不达标 | ⭐⭐⭐ | 增加气压计BMP280采集垂直高度特征；引入HMM时序建模利用活动转移概率平滑分类结果；设计代价敏感决策提升难分类别召回率 |
| 数据泄漏（标准化/特征选择在全量数据fit） | ⭐⭐⭐⭐ | 用`sklearn.Pipeline`封装预处理+特征选择+模型全流程，仅在训练折上fit参数；严格按`subject_id`划分训练/测试集（LOSO） |
| 焊接失败（虚焊/短路） | ⭐⭐⭐ | 优先采用“模块脚位”设计（GY-521/GY-273贴板），减少QFN芯片焊接难度；准备万用表测通断，用0.1mm漆包线飞线修复断路，刀片刮除短路 |

### 5.1 核心交付物

- 边缘推理：ESP32上单窗口推理时延≤100ms，满足实时性要求
