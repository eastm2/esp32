## 一、采集基本信息
| 项目 | 配置说明 |
|------|----------|
| 佩戴部位 | 右侧腰胯部，弹性绑带固定，传感器坐标系定义：X轴朝前、Y轴朝左、Z轴朝上 |
| 采样率 | 100Hz（满足奈奎斯特采样定理，可覆盖步行~2Hz、跑步~3Hz的主频特征） |
| 硬件方案 | 过渡期（D1-D5）：杜邦线+面包板连接ESP32+MPU6050+HMC5883L<br>正式期（D6起）：自制PCB扩展板（I2C地址：MPU6050=0x68，HMC5883L=0x1E） |
| 被试规模 | ≥3名健康成年被试，男女比例均衡，年龄18-25岁 |
| 活动类别 | 基础6类：**静坐、站立、步行、跑步、上楼、下楼**<br>进阶可选：跌倒（代价敏感场景）、异常静止 |

## 二、被试与数据划分规则（防泄漏核心）
### 2.1 被试独立划分（企业级红线）
严禁按“随机窗口”划分训练/测试集，必须按`subject_id`划分，从源头杜绝数据泄漏：
| 被试ID | 角色 | 说明 |
|--------|------|------|
| subject_01 | 训练集 | 所有窗口仅用于模型训练、调参、特征选择 |
| subject_02 | 训练集 | 同上 |
| subject_03 | 测试集 | 所有窗口仅用于最终模型评估，绝不参与训练/调参过程 |
| *后续新增被试* | 可按需加入训练/测试集，严格遵循“整被试划分”原则 |

### 2.2 单被试采集要求
每名被试每类活动采集≥3次，每次持续≥60s，两次采集间隔休息10s避免疲劳：
| 活动类别 | 单次采集时长 | 动作规范（避免歧义） |
|----------|--------------|----------------------|
| 静坐 | 60s | 自然坐于无扶手椅，上半身挺直，手臂自然放于大腿，不操作手机/电脑 |
| 站立 | 60s | 自然站立，双脚与肩同宽，手臂自然下垂，不倚靠墙面/家具 |
| 步行 | 60s | 平坦地面匀速行走，步频~2步/秒，手臂自然摆动，无跑跳/停顿 |
| 跑步 | 60s | 平坦地面匀速慢跑，步频~3步/秒，无变速/变向 |
| 上楼 | 60s | 步行楼梯逐级上行，一步一级台阶，不扶扶手、不跨级 |
| 下楼 | 60s | 步行楼梯逐级下行，一步一级台阶，轻扶扶手但不借力 |

## 三、采集流程规范
### 3.1 采集前准备（每日必做）
1. **硬件检查**：
   - 过渡期（杜邦线方案）：检查I2C接线是否牢固，上电后用I2C扫描工具确认0x68、0x1E设备可被识别
   - 正式期（PCB方案）：焊接完成后先测通断，上电后I2C扫描确认双传感器正常工作，无短路/虚焊
2. **传感器标定（D2完成后强制执行）**：
   采集前必须完成三项标定，保存标定参数至`calib_params/`目录：
   - 加速度计六位置标定（12参数最小二乘）
   - 磁力计椭球拟合（硬铁/软铁补偿）
   - 陀螺仪零偏估计
3. **佩戴固定**：确认传感器绑定位置与坐标系符合1.1要求，无松动风险。

### 3.2 单批次采集步骤
1. 启动ESP32采集程序，自动记录`timestamp`、`subject_id`、`hardware_type`（标记是杜邦线/PCB采集）
2. 被试完成1类动作的1次采集，过程中保持动作规范，无大幅体位突变
3. 采集结束后立即标注动作标签，保存为CSV文件，命名规则：`subject_{id}_{action}_{batch}.csv`（例：`subject_01_walk_01.csv`）
4. 采集下一批次前，确认传感器未发生移位，必要时重新固定。

### 3.3 过渡段处理（必执行）
每类动作采集的前后各2s为**过渡段**，标记为`transition`，不参与窗口切分：
- 例如：静坐开始前2s是“从站立到静坐”的过渡，结束后2s是“从静坐到站立”的过渡，直接裁剪丢弃，避免混合动作污染样本。

## 四、数据格式与存储规范
### 4.1 原始数据字段定义
所有CSV文件必须包含以下字段，新增字段需同步更新协议：
| 字段名 | 类型 | 说明 | 必选/可选 |
|--------|------|------|-----------|
| timestamp | int | 采样时间戳（单位：ms） | 必选 |
| ax/ay/az | float | 加速度计三轴原始读数（标定前为raw值，标定后为calibrated值，单位：g） | 必选 |
| gx/gy/gz | float | 陀螺仪三轴原始读数（标定后已减零偏，单位：°/s） | 必选 |
| mx/my/mz | float | 磁力计三轴原始读数（标定后已做硬铁/软铁补偿，单位：μT） | 必选 |
| label | string | 动作标签：sit/stand/walk/run/upstairs/downstairs/transition | 必选 |
| subject_id | string | 被试ID：subject_01/subject_02/subject_03 | 必选 |
| hardware_type | string | 硬件类型：dupont（杜邦线）/pcb_v1（自制PCB） | 必选 |
| pressure* | float | 气压值（仅PCB搭载BMP280时存在，单位：hPa） | 可选 |

### 4.2 目录结构规范
```
dataset/
├── raw/                      # 标定前原始数据（用于D8标定增益对比）
│   ├── subject_01/
│   │   ├── sit_01.csv
│   │   └── ...
│   └── ...
├── calibrated/               # 标定后数据（用于核心模型训练评估）
│   ├── subject_01/
│   └── ...
├── calib_params/             # D2生成的标定参数
│   ├── accel_calib.json      # 加速度计M矩阵+零偏b
│   ├── mag_calib.json        # 磁力计硬铁偏移+软铁变换矩阵
│   └── gyro_calib.json      # 陀螺仪零偏
└──采集记录表.xlsx            # 记录每名被试的采集时间、动作、批次、异常情况
```

## 五、窗口化与标签规则
### 5.1 滑动窗口参数
| 参数 | 取值 | 说明 |
|------|------|------|
| 窗口长度 | 2.56s | 覆盖1-2个完整步行周期，FFT频率分辨率≈0.39Hz，可分辨2Hz步行主频 |
| 重叠率 | 50% | 避免完整动作被窗口边界截断，保证特征有效性 |
| 窗口标签判定 | 窗口内≥80%样本属于同一类时才保留该窗口，否则标记为无效窗口并丢弃 | 防止混合动作样本干扰模型训练 |

### 5.2 禁止性规则（企业红线）
❌ 禁止将同一被试的连续窗口同时划分到训练集和测试集  
❌ 禁止在测试集上进行超参数调优  
❌ 禁止使用未标定的`raw/`数据直接训练最终模型  
❌ 禁止采集时传感器发生滑动、移位

## 六、PCB切换验证规则（冲80+必做）
1. PCB到货焊接完成并通过I2C扫描后，**必须重新采集subject_01的6类活动各1次**，保存为`hardware_type=pcb_v1`的数据
2. 对比同一被试、同一动作下，杜邦线方案与PCB方案的以下指标：
   - 标定后合加速度模长分布（应在0.99-1.01g之间）
   - 核心时域特征（均值、RMS、SMA）的分布差异（差异应<5%）
3. 验证通过后，方可使用PCB采集的数据替换杜邦线数据用于最终评估。

## 七、异常处理机制
| 异常场景 | 处理方式 |
|----------|----------|
| 采集过程中传感器移位 | 立即停止采集，该批次数据标记为无效并删除，重新固定后重新采集 |
| I2C通信中断/丢帧 | 检查接线/焊接质量，重新上电后重新采集该批次数据 |
| 磁力计读数异常（受WiFi干扰） | 采集时关闭ESP32的WiFi功能，确保HMC5883L远离天线≥15mm |
| 被试动作不规范 | 当场提醒纠正，若已采集则标记为无效，重新采集 |
