## 数据采集协议

### 1. 基本信息
- **被试编号**：Subject1、Subject2、Subject3
- **佩戴部位**：**腰部**（将传感器固定在腰部正中，X 轴指向身体前方，Y 轴垂直向上，Z 轴指向身体右侧，确保采集期间不松动）
- **采样率**：**50 Hz**（满足奈奎斯特定理，人体运动主要频率 ≤ 20 Hz）

### 2. 活动类别与采集方案
| 活动类别 | 每类每人重复次数 | 每次采集时长 | 说明 |
|----------|----------------|--------------|------|
| 静坐     | 3 次 | ≥ 60 秒 | 正常静坐，背部挺直或自然放松，避免大幅晃动 |
| 站立     | 3 次 | ≥ 60 秒 | 自然站立，允许微小重心调整，不可走动 |
| 步行     | 3 次 | ≥ 60 秒 | 平地直线匀速步行 |
| 跑步     | 3 次 | ≥ 60 秒 | 慢跑或中等速度跑，直线前进 |
| 上楼     | 3 次 | ≥ 60 秒 | 正常速度一步一级攀登楼梯，楼梯斜度适中 |
| 下楼     | 3 次 | ≥ 60 秒 | 正常速度一步一级下楼梯，楼梯斜度适中 |
| （可选）跌倒 | 3 次 | 单次约 10 秒，但标记前后过渡 | 在软垫上模拟前向/侧向跌倒，记录异常静止状态 |

**总数据量预估**：  
3 被试 × 6 类 × 3 次 × 60 s = 3240 秒（54 分钟）有效信号，50 Hz 下约 162,000 个采样点。

### 3. 标签协议
- **开始/结束打标**：每次采集前由实验员按压按键标记开始，结束时再次按压标记结束；同时在被试动作稳定开始 2 秒后手动添加“正式数据起点”，结束前 2 秒标记“终点”。
- **过渡段处理**：每次动作开始后的**前 2 秒**和结束前的**后 2 秒**自动裁除，仅保留中间稳定段作为有效样本。
- **标签记录方式**：将活动类别 ID 编码为整数：
  - 0: 静坐
  - 1: 站立
  - 2: 步行
  - 3: 跑步
  - 4: 上楼
  - 5: 下楼
  - （6: 跌倒）
- 所有标签与传感器数据通过时间戳对齐，写入 CSV 文件的 `label` 列。

### 4. 数据落盘格式
每条样本为一行 CSV 文本，列顺序如下：
```
timestamp, ax, ay, az, gx, gy, gz, mx, my, mz, label, subject_id
```
- `timestamp`: 毫秒级时间戳（相对采集开始）
- `ax, ay, az`: 加速度计三轴原始读数（g）
- `gx, gy, gz`: 陀螺仪三轴读数（°/s）
- `mx, my, mz`: 磁力计三轴读数（μT）
- `label`: 活动类别编号（0-5）
- `subject_id`: 被试编号（1、2、3）

### 5. 数据划分原则
- **训练/测试集划分**：严格按 `subject_id` 分层划分，例如采用留一被试法（Leave-One-Subject-Out, LOSO）：每次选择 2 名被试的数据作为训练集，剩余 1 名被试的数据作为测试集，共进行 3 次独立评估，最终取平均性能。
- **禁止数据泄漏**：同一被试的所有窗口必须完整出现在同一集合中，绝不允许随机打散后按比例划分。
- **窗口内防泄漏**：当使用重叠滑动窗口时，确保训练/测试集的窗口来源于不同的原始连续信号段，避免信息通过时间邻近窗口泄漏。本项目在划分时直接按被试分离，窗口在各自被试内独立生成。
